# Анализ Stable Clear v2.1 и план исправления v2.2

## Подтверждённый результат v2.1

По игровому тесту упорядоченный путь `async=0`, `cpuSync=1` больше не вызывает
`DXGI_ERROR_DEVICE_REMOVED`, а FPS приемлем. Это позволяет зафиксировать
синхронизацию v2.1 и не менять её в графическом исправлении.

Лог профиля Reference Quality показывает, что стабильность сохраняется даже при
`12/12` shadow samples, `AO=24`, `sky=8` и десятках источников. Следовательно,
оставшиеся точки/полосы и вращающиеся тени не являются просто следствием слишком
малого количества samples.

## 1. Почему тень поворачивается вместе с камерой

Fallback-свет создавался относительно камеры:

```cpp
VectorCopy(backEnd.refdef.vieworg, lightOrg);
VectorMA(lightOrg, 128.0f, backEnd.refdef.viewaxis[0], lightOrg);
VectorMA(lightOrg, 72.0f, backEnd.refdef.viewaxis[2], lightOrg);
```

Это было полезно для тёплой объёмной картинки Clean Reference, но такой источник
не является частью карты. В v2.1 он участвовал в `TraceSoftShadow`, поэтому при
повороте камеры перемещалась позиция света и менялось направление тени.

Исправление v2.2: оставить fallback как fill/specular light, но пометить его
`fill-only`. Он освещает сцену, но не трассирует тени и не заменяет shadow mask
настоящих источников карты. Отдельный cvar
`r_dxrFallbackCastsShadows 1` сохраняет старое поведение только для диагностики.

## 2. Почему видны чёрные точки, сетки и белые полосы

### Недостаточная точность world position

В v2.1 абсолютные координаты мира записывались в:

`DXGI_FORMAT_R16G16B16A16_FLOAT`

У half-float только 10 бит мантиссы. При координатах карты в сотнях и тысячах
единиц соседние поверхности получают грубо квантизованные позиции. Теневой луч
стартует то над, то под реальной поверхностью, что даёт self-intersection,
shadow acne, точки и полосы, особенно при движении камеры.

Исправление v2.2:

`DXGI_FORMAT_R32G32B32A32_FLOAT`

для position render target, SRV и обоих graphics PSO.

### Потерянное отсечение self-intersection

Старый Clean Reference применял для shadow rays:

`RAY_FLAG_CULL_FRONT_FACING_TRIANGLES`

В v2 оно было удалено. v2.2 возвращает этот флаг вместе с
`RAY_FLAG_FORCE_OPAQUE`, чтобы луч, сдвинутый от поверхности, не принимал
исходный polygon за occluder.

### Невалидный G-buffer

ARB/color-only pipeline пишет raster color, но его write mask для normal,
position и velocity равен нулю. В таких пикселях DXR видел depth от геометрии,
но position/normal оставались очищенными нулями.

Исправление v2.2: в `position.w` записывается `geometryFlag + 1`; ноль становится
явным признаком отсутствия G-buffer. Такие пиксели сохраняют raster albedo и не
проходят через ошибочное RT lighting.

### Искусственный screen-space UV jitter

Fixed-function HLSL v2.1 добавлял к UV шум, зависящий от `SV_Position`. При
движении камеры одна и та же поверхность выбирала другие texels, что усиливало
мерцание, диагональные структуры и полосы.

Исправление v2.2: UV используются без screen-space смещения.

## 3. Почему Reference Quality не исправил картинку

Увеличение samples уменьшает Monte-Carlo noise, но не исправляет:

- перемещающийся камерный источник;
- FP16-квантизацию world position;
- self-intersection;
- неверные нулевые G-buffer данные;
- UV, меняющиеся вместе с экранной координатой.

Поэтому профили `1/1 AO=0 sky=0` и `12/12 AO=24 sky=8` могли показывать похожую
структурную сетку.

## 4. Текстуры

v2.2 сохраняет:

- `r_picmip 0`;
- `r_picmip2 0`;
- `r_roundImagesDown 0`;
- `r_simpleMipMaps 0`;
- `r_texturebits 32`;
- trilinear mip filtering;
- 8x anisotropic filtering в D3D12 root signature.

Это улучшает резкость на наклонных поверхностях и исключает искусственное
снижение разрешения. Но исходные RTCW assets остаются старыми текстурами своего
реального размера. Настройки фильтрации не могут создать детали, которых нет в
файле; для настоящих 2K/4K материалов нужен отдельный texture pack.

## 5. Что не является основной причиной этих артефактов

Повторяющиеся сообщения:

`R_AddMDCSurfaces: no such frame ...`

касаются некорректных animation frame indices некоторых MD3/MDC-моделей. Их
нужно исправлять отдельно. Они не объясняют неподвижные точечные узоры на всех
стенах и полу и не меняют позицию fallback-света.

## Статус проверки v2.2

Проверено статически:

- последовательное применение `patch 10 → patch 20 v2.1 → patch 30 v2.2`;
- ожидаемые Git blob hashes v2.1;
- `git apply --check` и reverse check;
- `git diff --check`;
- workflow, profile names и структура runtime package;
- наличие source markers для каждого исправления.

Не проверено в этой среде:

- компиляция MSVC/DXC;
- работа на конкретном D3D12/DXR-драйвере;
- фактическое исчезновение артефактов в игре.

Эти пункты подтвердят GitHub Actions и следующий игровой тест.
