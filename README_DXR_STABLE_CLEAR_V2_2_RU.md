# DarkWolf RTCW DXR Stable Clear v2.2 — визуальная корректность

## Зачем нужен v2.2

Stable Clear v2.1 подтвердил главное: синхронный упорядоченный путь стабилен и
даёт приемлемый FPS. Скриншоты v2.1, однако, показали отдельные ошибки
корректности рендера, которые не устраняются увеличением числа лучей:

1. Камерный fallback-свет перемещался вместе с `vieworg/viewaxis` и сам бросал
   тени. Поэтому направление «теней от факела» менялось при повороте камеры.
2. Абсолютная позиция мира хранилась в `R16G16B16A16_FLOAT`. Для координат
   большой RTCW-карты точности FP16 недостаточно; начало теневого луча
   квантуется, что проявляется точками, полосами и shadow acne.
3. Некоторые ARB/color-only проходы не заполняют normal/position G-buffer. DXR
   принимал очищенные нули за настоящую геометрию.
4. Fixed-function shader намеренно сдвигал UV на экранный шум. При движении
   камеры это меняло выбранный texel и усиливало мерцание/полосы.

## Что меняет patch 30

- Камерный fallback остаётся тёплой заполняющей подсветкой для сохранения
  подтверждённого вида Clean Reference, но по умолчанию не трассирует тени.
- Реальные факелы и потолочные лампы продолжают бросать RT-тени из своих
  мировых координат.
- Position G-buffer переведён в `R32G32B32A32_FLOAT`.
- Для shadow rays восстановлено отсечение front-facing self-intersections.
- Добавлен признак валидности G-buffer; неподдерживаемые color-only пиксели
  сохраняют обычный raster color.
- Удалён screen-space UV jitter.
- D3D12 samplers используют 8x anisotropic filtering.
- Исправлен приоритет выбора света: огромный приоритет получает только явно
  помеченный fallback-fill, а не каждый динамический источник карты.

## Что v2.2 не может сделать

`r_picmip 0`, исходное разрешение и anisotropic filtering сохраняют максимум
деталей, уже присутствующих в RTCW. Они не превращают старую 256×256 текстуру в
настоящую 2K/4K-текстуру. Для новых деталей потребуется отдельный texture pack.

## Установка поверх существующего репозитория v2.1

Загрузите **содержимое** папки `repo-overlay` в корень репозитория и сделайте
commit. Старый workflow v2.1 можно оставить: новый workflow его не запускает.

Запустите в GitHub Actions:

`DarkWolf DXR Stable Clear v2.2 Visual Correctness`

Конфигурация: `Release`.

Artifact:

`DarkWolf-DXR-Stable-Clear-v2.2-Release`

Устанавливайте artifact в отдельную чистую копию игры.

## Порядок теста

1. `RUN_DXR_STABLE_CLEAR_V22.bat` — основной профиль.
2. На том же месте медленно поворачивайте камеру вокруг факела/лампы. Форма и
   направление геометрической тени должны оставаться привязанными к источнику.
3. `RUN_DXR_STABLE_REAL_LIGHTS_V22.bat` — полностью отключает камерный fill и
   служит контрольным тестом реальных источников карты.
4. Проверьте пол, стены под острым углом и движение камеры. Крупные чёрные точки,
   диагональные сетки и белые полосы должны исчезнуть или резко уменьшиться.
5. `RUN_DXR_REFERENCE_QUALITY_V22.bat` — только после основного теста; это
   дорогой профиль сравнения, а не рекомендуемый игровой режим.

Лог исправной версии должен начинаться с:

`DXR v2.2 visual-correct:`

и показывать:

`async=0 cpuSync=1 fallback=1 fallbackShadows=0`

## Ограничение проверки

Patch 30 статически проверен на точной последовательности `patch 10 → patch 20
v2.1 → patch 30 v2.2`. Реальную компиляцию MSVC/DXC и поведение на видеокарте
подтвердит только GitHub Actions и игровой тест.
