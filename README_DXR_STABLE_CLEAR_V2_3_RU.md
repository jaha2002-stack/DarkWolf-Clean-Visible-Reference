# DarkWolf RTCW DXR Stable Clear v2.3 — Shadow Composite

Эта версия является небольшим графическим обновлением поверх подтвержденной
стабильной базы v2.2. Синхронизация, fences, формат position G-buffer,
фильтрация текстур и защита от DEVICE_REMOVED не изменяются.

## Подтвержденный диагноз

В v2.2 диагностическая маска `debugMode 5` показывает силуэт прутьев решетки и
тени факелов. Следовательно, геометрия присутствует в TLAS, теневые лучи
перекрываются корректно и источник проблемы находится в финальном composite.

Старая формула применяла authored-light shadow только к legacy-части:

```hlsl
lerp(rtLitColor, baseAlbedo * legacyShadow, legacyBlend)
```

Яркий camera-fill, ambient и RT diffuse оставались незатененными и почти
полностью скрывали корректную маску.

## Что меняет patch 40

- разделяет RT diffuse и specular;
- смешивает RT diffuse с raster base без тени;
- после смешивания применяет authored-light shadow ко всей diffuse-части;
- specular затемняется только частично, чтобы сохранить металлические блики;
- повышает чувствительность composite к вкладу реального источника;
- не включает тени от camera fallback, поэтому вращающиеся вместе с камерой
  тени не возвращаются.

## Первый запуск

Используйте:

```text
RUN_DXR_STABLE_SHADOWS_V23.bat
```

Если тени покажутся слишком слабыми:

```text
RUN_DXR_STABLE_SHADOWS_STRONG_V23.bat
```

Для проверки только настоящих источников карты:

```text
RUN_DXR_STABLE_REAL_LIGHTS_V23.bat
```

Белая/серая диагностическая маска:

```text
RUN_DXR_SHADOW_MASK_V23.bat
```

В строке диагностики должно быть:

```text
DXR v2.3 shadow-composite:
async=0 cpuSync=1 fallbackShadows=0
```

## Ожидаемый результат

- тень открывающейся решетки должна быть видна в обычной цветной картинке;
- кронштейн факела и другие препятствия должны давать более отчетливые тени;
- направление теней не должно поворачиваться вместе с камерой;
- стабильность должна остаться такой же, как в v2.2.

Реальная сборка MSVC/DXC и результат на GPU требуют проверки через GitHub
Actions и игровой тест.
